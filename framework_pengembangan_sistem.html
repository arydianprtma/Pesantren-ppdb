6. Pola Desain yang Diterapkan

6.1 Events & Observers Pattern

Sistem menggunakan Event-Driven Architecture untuk pemrosesan asinkron dan notifikasi real-time:

SpmbPendaftaran dibuat / diperbarui
    → Event: SpmbPendaftaranCreated / SpmbPendaftaranUpdated
        → Queue Worker memproses event
            → Broadcast via Laravel Reverb (WebSocket)
                → Frontend menerima notifikasi real-time (Laravel Echo)

Observer digunakan di app/Observers/ untuk memantau perubahan Model secara otomatis tanpa mencemari logika Controller.

6.2 Caching Strategy

Konten statis di-cache selama 1 jam (3600 detik) menggunakan Cache::remember() untuk mempercepat response time:

Cache KeyKontenDurasi
home_prestasiDaftar 6 prestasi terbaru1 jam
home_berita3 berita terbaru1 jam
home_visi_misiVisi misi aktif1 jam
fasilitasSeluruh fasilitas aktif1 jam
sejarahSeluruh konten sejarah1 jam

6.3 SPA via Inertia.js

Inertia.js menghilangkan kebutuhan REST API terpisah. Laravel Controller merender halaman Vue.js secara langsung menggunakan Inertia::render('PageName', $data), sehingga:

  • Navigasi antar halaman tidak memuat ulang browser penuh (Single Page Application)
  • Data dikirim dari server ke Vue.js sebagai props
  • Tetap menggunakan routing, middleware, dan session Laravel (bukan JWT)

8. Alur Autentikasi Sistem

Pengguna → POST /login (email + password)
    → Laravel Breeze memvalidasi kredensial
        → Laravel Sanctum membuat session
            → Redirect ke dashboard

Akses /admin (Filament Panel)
    → Filament memeriksa session & role pengguna
        → canAccessPanel() diperiksa:
            - is_active = true  ✓
            - role ∈ [super_admin, admin, editor]  ✓
        → Tampilkan Panel Admin  |  Redirect 403 Forbidden

10. Kesimpulan Framework

Sistem Informasi Pesantren & SPMB ini dibangun menggunakan pendekatan Full-Stack Laravel dengan stack teknologi modern yang dipilih secara strategis:

AspekPilihan TeknologiAlasan
BackendLaravel 12Framework PHP terpopuler, ekosistem lengkap, keamanan bawaan
FrontendVue.js 3 + Inertia.jsReaktif, SPA feel tanpa REST API terpisah
Admin PanelFilament v5CRUD otomatis, hemat waktu pengembangan
Real-timeLaravel ReverbSelf-hosted WebSocket, tanpa biaya pihak ketiga
StylingTailwindCSSUtility-first, responsif, konsisten
DatabaseSQLite → MySQLFleksibel sesuai lingkungan
Build ToolVite v7Cepat, mendukung HMR untuk pengembangan
Metode Pengembangan: Sistem dikembangkan secara iteratif — fitur dibangun bertahap dimulai dari modul portal informasi, dilanjutkan modul SPMB, dan diakhiri dengan panel admin terintegrasi. Setiap iterasi menghasilkan modul yang dapat langsung digunakan (incremental delivery).
